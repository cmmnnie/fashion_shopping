# 장바구니 페이지 문서

## 1. 기본 정보

- **문서명**: 장바구니 페이지 문서
- **프로젝트명**: Fashion Shopping
- **참여한 팀원**: 차민희
- **페이지 URL**
  - 메인 상품 목록 페이지: `http://localhost:3000/`
  - 장바구니 페이지: `http://localhost:3000/cart`
- **Storybook URL**
  - Storybook 메인: `http://localhost:6006`
  - 결제하기 버튼 문서 페이지: `http://localhost:6006/?path=/docs/components-checkoutbutton--docs`

> 현재 URL은 로컬 실행 기준입니다. 배포 URL이 있다면 추후 해당 주소로 교체할 수 있습니다.

---

## 2. 개요

장바구니 페이지는 사용자가 선택한 상품을 확인하고,  
수량을 조정하며, 배송비와 최종 결제 금액을 확인한 뒤  
결제를 진행할 수 있도록 구성한 화면입니다.

기존 상품 목록 페이지와 결제 모듈 사이를 연결하는 핵심 페이지이며,  
사용자의 구매 흐름에서 중요한 역할을 합니다.

## 4. 설치 및 실행 방법

### 4-1. 프로젝트 실행

프로젝트 루트에서 아래 명령어를 순서대로 실행합니다.

```bash
cd frontend
npm install
npm start

실행 후 브라우저에서 아래 주소로 접속합니다.

http://localhost:3000

컴포넌트 문서화를 확인하기 위해 아래 명령어를 실행합니다.

실행 후 브라우저에서 아래 주소로 접속합니다.

http://localhost:6006

4-3. 테스트 실행

기본 테스트를 진행할 경우 아래 명령어를 사용할 수 있습니다.

5. 페이지 구성 및 주요 기능
5-1. 사용자 흐름
1.사용자가 메인 상품 목록 페이지에서 상품을 선택합니다.
2.장바구니 페이지로 이동하여 담긴 상품을 확인합니다.
3.상품 수량을 늘리거나 줄일 수 있습니다.
4.필요 없는 상품은 삭제할 수 있습니다.
5.상품 총액, 배송비, 최종 결제 금액을 확인합니다.
6.결제하기 버튼을 눌러 결제 창을 확인합니다.

5-2. 주요 기능
-장바구니 상품 목록 표시
-상품 수량 증가/감소
-상품 삭제
-상품 총액 계산
-배송비 계산
  -100,000원 이상 무료배송
  --100,000원 미만 3,000원
-최종 결제 금액 표시
-결제 모달 연결

6. 컴포넌트 구조

장바구니 페이지는 다음과 같은 주요 컴포넌트로 구성되어 있습니다.

App
-장바구니 전체 상태를 관리합니다.
-상품 목록, 총액, 배송비, 결제 모달 상태를 제어합니다.

CartList
-장바구니에 담긴 여러 상품을 목록으로 표시합니다.
-상품이 없을 경우 빈 상태 메시지를 보여줍니다.

CartItem
-장바구니 상품 1개의 정보를 표시합니다.
-상품 이미지, 이름, 가격, 수량, 금액을 보여줍니다.
-수량 증가/감소 버튼과 삭제 버튼을 포함합니다.

PaymentModal
-결제하기 버튼을 눌렀을 때 열리는 결제 창입니다.
-상품 총액, 배송비, 최종 결제 금액을 보여줍니다.
-닫기 및 결제 완료 동작을 제공합니다.

CheckoutButton / Button
-결제를 시작하는 버튼입니다.
-장바구니가 비어 있으면 비활성화됩니다.

7. 금액 계산 기준
-상품 총액 = 각 상품 가격 × 수량의 합
-배송비
  -100,000원 이상: 0원
  -100,000원 미만: 3,000원
-최종 결제 금액 = 상품 총액 + 배송비

8. 테스트 방법
8-1. 수동 테스트 절차

아래 순서대로 직접 확인합니다.

1. npm start 실행 후 http://localhost:3000/cart 접속
2. 장바구니 상품이 목록 형태로 보이는지 확인
3. + 버튼 클릭 시 수량이 증가하는지 확인
4. - 버튼 클릭 시 수량이 감소하는지 확인
5. 삭제 버튼 클릭 시 상품이 제거되는지 확인
6. 상품 총액이 수량 변화에 맞게 바뀌는지 확인
7. 총액이 100,000원 이상/미만일 때 배송비가 올바르게 표시되는지 확인
8. 결제하기 버튼 클릭 시 결제 모달이 열리는지 확인

8-2. Storybook 확인 방법
1. npm run storybook 실행
2. http://localhost:6006 접속
3. 좌측 메뉴에서 문서화된 컴포넌트 확인
예시 링크:
- 결제하기 버튼 문서: http://localhost:6006/?path=/docs/components-checkoutbutton--docs
  로컬 환경에서 실행 중이어야 페이지 확인이 가능합니다.

  9. 기능 테스트 결과

아래 항목은 현재 구현 내용과 이전 테스트 결과를 기준으로 정리한 것입니다.

1) 뒤로가기 버튼을 이용해 메인 상품 목록 페이지로 이동할 수 있나요?
-현재 상태: 미구현 또는 별도 확인 필요
-현재까지 공유된 구현 흐름에서는 장바구니 목록, 수량 조절, 삭제, 금액 계산, 결제 모달 중심으로 확인되었습니다.
-뒤로가기 버튼이 실제 화면에 구현되어 있다면 http://localhost:3000/cart에서 클릭 후 http://localhost:3000/로 이동하는지 별도 확인이 필요합니다.

2) 결제하기 버튼을 누르면, 결제할 수 있는 창이 보이나요?
-현재 상태: 동작 확인 완료
-결제하기 버튼 클릭 시 결제 모달이 열리도록 구현되어 있습니다.

3) 카드 등록 및 결제가 가능한가요?
-현재 상태: 실제 결제 API 연동 전 단계
-현재는 결제 모달이 열리고 결제 완료 버튼을 누르는 흐름까지는 확인되었으나,
 실제 카드 등록 및 외부 결제 연동은 아직 구현되지 않았습니다.
-따라서 실제 카드 결제는 불가능하며, 결제 UI 흐름만 확인 가능한 상태입니다.

10. 유의사항 및 알려진 이슈
-현재 페이지 URL은 로컬 실행 기준(localhost)입니다.
-결제 모달은 UI 흐름 확인용이며, 실제 결제 API는 아직 연결되지 않았습니다.
-카드 등록 및 실제 결제 처리는 미구현 상태입니다.
-뒤로가기 버튼 기능은 현재 문서 기준에서 별도 구현 확인이 필요합니다.
-Storybook 링크는 npm run storybook 실행 상태에서만 정상 확인할 수 있습니다.
