이 문서는 이후 개발 작업을 계획하고 칸반 보드에 작업을 등록하는 기준으로 사용됩니다.

---

## 2. 요구사항 개요

기존 시스템에서는 사용자가 상품을 선택하면 바로 결제 페이지로 이동하는 구조였습니다.

고객사는 사용자가 여러 상품을 비교하고 고민할 수 있도록 장바구니 페이지를 추가하기를 요청하였습니다.

장바구니 페이지에서는 사용자가 담은 상품을 확인하고, 금액을 확인한 후 결제를 진행할 수 있도록 하는 것이 목적입니다.

---

## 3. 고객 요구사항

고객사는 다음과 같은 기능을 요청하였습니다.

### 3.1 장바구니 페이지 추가

- 상품 선택 시 바로 결제 페이지로 이동하지 않고 장바구니에 상품을 담을 수 있어야 합니다.
- 사용자는 장바구니 페이지에서 담은 상품 목록을 확인할 수 있습니다.

---

### 3.2 상품 목록 표시

장바구니 페이지에서는 다음 정보가 표시되어야 합니다.

- 상품 이미지
- 상품 이름
- 상품 가격
- 상품 수량

사용자는 장바구니에서 현재 선택한 상품들을 확인할 수 있어야 합니다.

---

### 3.3 상품 수량 표시 및 변경

사용자는 장바구니에서 상품 수량을 확인하고 변경할 수 있어야 합니다.

요구사항

- 상품 수량 표시
- 수량 증가 기능
- 수량 감소 기능
- 수량 변경 시 금액 자동 반영

---

### 3.4 상품 금액 계산

장바구니 페이지에서는 상품 총액이 정확하게 계산되어 표시되어야 합니다.

계산 기준

- 상품 가격
- 상품 수량

상품 금액은 수량 변경 시 자동으로 업데이트되어야 합니다.

---

### 3.5 배송비 정책

배송비는 다음 정책을 적용합니다.

- 총 구매 금액이 100,000원 이상일 경우 배송비 무료
- 총 구매 금액이 100,000원 미만일 경우 배송비 3,000원

배송비는 상품 총액을 기준으로 계산되며 화면에 표시되어야 합니다.

---

### 3.6 총 결제 금액 표시

장바구니 페이지에서는 다음 금액 정보가 표시되어야 합니다.

- 상품 총액
- 배송비
- 최종 결제 금액

최종 결제 금액은 다음 공식으로 계산됩니다.


---

### 3.7 뒤로가기 버튼

장바구니 페이지에는 뒤로가기 버튼이 제공되어야 합니다.

요구사항

- 뒤로가기 버튼 클릭 시 상품 목록 페이지로 이동

---

### 3.8 결제 기능 연동

장바구니 페이지에는 결제하기 버튼이 제공되어야 합니다.

요구사항

- 결제하기 버튼 클릭 시 기존 결제 모듈 실행
- 결제 모듈은 모달 형태로 표시

---

## 4. 화면 구성 요소

장바구니 페이지는 다음과 같은 영역으로 구성됩니다.

- 장바구니 상품 목록 영역
- 상품 수량 조절 영역
- 금액 계산 영역
- 배송비 정보 영역
- 결제 버튼 영역

## 5. 요구사항 핵심 포인트

장바구니 기능 구현 시 다음 항목이 중요합니다.

- 상품 목록 표시
- 상품 수량 관리
- 상품 총액 계산
- 배송비 정책 적용
- 최종 결제 금액 계산
