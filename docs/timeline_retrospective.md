# 📅 프로젝트 타임라인 회고

## 1. 개요
본 문서는 쇼핑몰 프로젝트 진행 과정에서의 주요 사건들을 시간 순서에 따라 정리하고,  
그 과정에서 배운 점과 의미 있었던 경험을 돌아보기 위해 작성되었습니다.

---

## 2. 주요 타임라인

### 🔹 초기 단계
- 프로젝트 저장소 생성 및 README 작성
- 상품 목록 페이지 구현 시작

### 🔹 기능 확장 단계
- 장바구니 기능 구현
- 결제 모달 및 결제 완료 페이지 추가

### 🔹 구조 개선 단계
- 상태 관리 구조 개선 (App 중심으로 통합)
- 컴포넌트 분리 및 리팩터링 진행

### 🔹 서비스 흐름 완성 단계
- React Router 도입
- 상품 목록 → 상세 → 장바구니 → 결제 흐름 연결

### 🔹 기능 확장
- 상품 상세 페이지 구현
- 페이지 간 데이터 일관성 확보

### 🔹 문서화 및 테스트
- Storybook 도입 및 UI 문서화
- Cypress 테스트 환경 구축

### 🔹 마무리 단계
- 고객용 문서 작성
- 개발자 가이드 정리
- 최종 기능 점검 및 프로젝트 정리

---

## 3. 주요 기술 학습 흐름

- React 컴포넌트 구조 이해 및 적용
- 상태 관리 개념 학습 및 구조 개선 경험
- Router를 통한 페이지 연동 구현
- Storybook을 통한 UI 문서화 경험
- Cypress를 통한 테스트 자동화 경험
- Git 커밋 전략 및 문서 관리 방법 학습

---

## 4. 인상 깊었던 경험 3가지

### 1. 상태 관리 구조 개선
장바구니 데이터를 상위 컴포넌트에서 관리하도록 변경하여  
페이지 간 데이터 일관성을 확보함

---

### 2. 사용자 흐름 중심 설계
단순 기능 구현이 아닌  
상품 → 상세 → 장바구니 → 결제 흐름 중심으로 설계

---

### 3. 고객 중심 문서 작성
개발자 관점이 아닌  
비개발자도 이해할 수 있도록 기능 중심 문서 작성

---

## 5. 회고

이번 프로젝트는 단순 기능 구현을 넘어,  
구조 설계, 사용자 흐름, 문서화까지 경험한 프로젝트였습니다.

특히 데이터 일관성과 사용자 경험을 고려하는 과정에서  
개발자로서 한 단계 성장할 수 있었습니다.
